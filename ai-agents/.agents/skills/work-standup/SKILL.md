# Work Standup

Публикует ответ на daily-бот команды в TiMe (dws-ai-team). Читает планы из vault, формирует черновик standup и отправляет в тред после подтверждения пользователя.

## Канал и бот

- **Канал:** `dws-ai-team` (id: `jpywmth1z3bjpccf1bsc78k7ha`)
- **Бот:** `dws_ai_bot`
- **Расписание бота:** Mon, Tue, Wed, Fri (не Thu)

## Шаблон standup

Бот задает 4 вопроса. По умолчанию заполняем только блоки «Вчера» и «Сегодня».
Пункты 3 (обсуждение на синке) и 4 (блокеры) - только если есть что сказать.

Строгий формат сообщения:

```
**Вчера:**
* [DWSAI-NNN](https://jira3.tcsbank.ru/browse/DWSAI-NNN): <суть> — <статус> [MR !NN](https://gitlab.tcsbank.ru/dwsai/dwsai-data-agent/-/merge_requests/NN)

* <пункт без задачи>


**Сегодня:**
* Разобрать ревью [MR !NN](url) ([DWSAI-NNN](url))

* <пункт>
```

Правила форматирования:
- Заголовки `**Вчера:**` и `**Сегодня:**` — жирный текст, без нумерации
- Буллеты через `*` (не `-`)
- Пустая строка между каждым пунктом внутри блока
- Двойная пустая строка между блоками «Вчера» и «Сегодня»
- Всегда ссылки на JIRA и GitLab MR — не пиши номера без ссылок

## Алгоритм

### 1. Найди последнее сообщение бота

Прочитай последние 30 сообщений канала `dws-ai-team`:
```
read_channel(channel_id="jpywmth1z3bjpccf1bsc78k7ha", limit=30)
```
Найди последний пост с `author == "dws_ai_bot"`. Запомни его `id` — это `root_id` для ответа в тред.

Если поста нет или он старше 2 дней — не отправляй, сообщи пользователю.

### 2. Прочитай вчерашний daily note

Файл: `1_Planning/Daily/2026/<вчерашняя дата>.md`

Источник для блока «Вчера»:
- Секция `## Done` внутри `<!-- daily-log -->` — если заполнена (факты из сессии)
- Иначе — отмеченные `[x]` задачи из `## Delivery` и `## Ops`

Держи кратко: 2-4 пункта, без деталей MR-описания. См. шаблон выше.

### 3. Прочитай сегодняшний daily note

Файл: `1_Planning/Daily/2026/<сегодняшняя дата>.md`

Источник для блока «Сегодня»:
- Незачеркнутые задачи из `## Delivery`
- Значимые задачи из `## Ops` (если есть внешний эффект, например груминг с дедлайном)

Держи кратко: 3-5 пунктов. Ставь ссылки на JIRA и GitLab, если задача или MR упоминаются.

### 4. Покажи черновик и отправь

**Интерактивный режим** (по умолчанию): выведи итоговый текст, явно попроси подтверждение — не отправляй без него.

**Авто-режим** (scheduled / `--auto`): подтверждение не требуется, отправь сразу. Используй, когда skill запускается по расписанию без участия пользователя.

### 5. Отправь в тред

```
create_post(root_id="<id поста бота>", message="<текст standup>")
```

## Safety

- В интерактивном режиме никогда не отправляй без явного подтверждения пользователя.
- В авто-режиме всегда логируй факт отправки (пост отправлен / ошибка).
- Не выдумывай задачи и результаты — только то, что есть в daily notes.
- Если daily note за вчера не найдена — скажи об этом, не составляй блок «Вчера» из головы.
- Если сообщение от бота не найдено — не постить в канал напрямую, уточнить у пользователя.

# Python Docstring Editor

## Задача

Обработай Python-файлы по пути, переданному в `$ARGUMENTS`.

Измени файлы на диске: улучши docstrings и, только когда тип надёжно следует из кода, добавь или уточни type annotations.

## Правила редактирования

- Работай только с `.py`-файлами внутри переданного пути.
- Не изменяй `.venv/`, `venv/`, `__pycache__/`, `build/`, `dist/`, `.git/` и сгенерированные файлы.
- Не меняй бизнес-логику, алгоритмы, значения констант, порядок исполнения, публичные контракты и строки runtime-событий.
- Не выполняй рефакторинг вне docstrings и очевидных type annotations.
- Сохраняй существующий стиль форматирования, импортов и кавычек.

## Правила docstrings

Используй Google Python Style Guide.

- Текст docstrings пиши на русском.
- Заголовки секций сохраняй на английском: `Args`, `Returns`, `Raises`, `Yields`, `Attributes`, `Examples`.
- Для модуля кратко опиши назначение и границу ответственности.
- Для класса добавляй `Attributes`, если поля являются частью понятного контракта.
- Для функции или метода добавляй только полезные секции.
- Не описывай исключения или поведение, которые нельзя подтвердить по коду.
- Убирай воду, повторы, канцелярит и лишние англицизмы; общие правила языка и обработки англицизмов — skill `russian-technical-editor`.
- Сохраняй технические идентификаторы и закреплённые термины: имена классов, API, DTO, fields, `skill`, `capability`, `runtime preflight`, `SKILL.md`, `thread_id` и подобные значения.

## Правила type annotations

- Добавляй annotations только там, где тип однозначен из реализации или уже используемых моделей.
- Используй современный синтаксис Python: `str | None`, `list[str]`, `tuple[str, ...]`.
- Не усложняй typing без необходимости.
- Не меняй runtime-поведение ради типизации.

## Порядок работы

1. Определи все `.py`-файлы внутри переданного пути.
2. Прочитай каждый файл до редактирования.
3. Отредактируй каждый применимый файл на диске.
4. После завершения перечисли:
   - изменённые пути;
   - файлы, которые не потребовали правок;
   - места, где annotations не были добавлены из-за неоднозначного типа.
5. Не вставляй полный код файлов в ответ, если пользователь отдельно этого не запросил.
